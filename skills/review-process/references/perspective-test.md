# テスト項目レビュー観点

テストケース・テスト仕様書をレビューする際に確認すべき観点の一覧。全項目を網羅的にチェックすること。

## 1. 要件・設計とのトレーサビリティ

| # | チェック項目 |
|:--|:--|
| 1.1 | テストケースが対応する要件・設計項目を明示的に参照しているか |
| 1.2 | すべての要件に対してテストケースが存在するか（漏れがないか） |
| 1.3 | 要件の受け入れ基準がテスト期待値に反映されているか |
| 1.4 | 設計変更がテストケースに反映されているか |

## 2. テストケースの網羅性

| # | チェック項目 |
|:--|:--|
| 2.1 | 正常系テストが十分に定義されているか |
| 2.2 | 異常系テスト（無効入力、エラー条件）が網羅されているか |
| 2.3 | 境界値分析が適用されているか（最小値、最大値、境界±1） |
| 2.4 | 同値分割が適切に行われているか（各同値クラスから代表値を選択） |
| 2.5 | 組み合わせテストが必要な箇所で考慮されているか |
| 2.6 | 状態遷移テスト（該当する場合）が定義されているか |

## 3. テストケースの記述品質

| # | チェック項目 |
|:--|:--|
| 3.1 | 前提条件が具体的かつ再現可能に記述されているか |
| 3.2 | テスト手順が第三者でも実行可能な粒度で記述されているか |
| 3.3 | 期待結果が曖昧でなく、客観的に判定可能か |
| 3.4 | 後処理（データクリーンアップ等）が必要な場合に明記されているか |
| 3.5 | テストケース間の依存関係（実行順序制約）が明示されているか |

## 4. テストデータ

| # | チェック項目 |
|:--|:--|
| 4.1 | テストデータが具体的に定義されているか（「適当な値」などの曖昧表現はないか） |
| 4.2 | テストデータが要件の制約（桁数、型、値域）に準拠しているか |
| 4.3 | 境界値データが正確に設定されているか |
| 4.4 | テストデータの準備手順が明確か |
| 4.5 | 個人情報・機密情報がテストデータに含まれていないか |

## 5. テスト環境・前提

| # | チェック項目 |
|:--|:--|
| 5.1 | テスト実行に必要な環境条件が明記されているか |
| 5.2 | 外部システムのモック/スタブの要否が定義されているか |
| 5.3 | テスト実行のための事前データ投入が定義されているか |
| 5.4 | 並行実行時の影響（排他制御、データ競合）が考慮されているか |

## 6. 非機能テスト

| # | チェック項目 |
|:--|:--|
| 6.1 | パフォーマンステスト（応答時間、処理件数の閾値）が定義されているか |
| 6.2 | セキュリティテスト（認証、認可、インジェクション）が含まれているか |
| 6.3 | ユーザビリティテスト（該当する場合）が含まれているか |
| 6.4 | 回帰テスト範囲が明確に定義されているか |

## 7. テスト分析手法の適用

| # | チェック項目 |
|:--|:--|
| 7.1 | デシジョンテーブルが複雑な条件分岐に適用されているか |
| 7.2 | 直交表・ペアワイズテストが多パラメータの組み合わせに適用されているか |
| 7.3 | 原因結果グラフが必要な箇所で活用されているか |
| 7.4 | 選択したテスト手法の適用理由が説明されているか |
